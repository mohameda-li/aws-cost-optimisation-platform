Onboarding: Customer deployment bundle

In short
• You complete one application form with your AWS region, report schedule and chosen services.
• Once approved, you download a ready-made deployment bundle for your AWS account.
• After deployment, reports run automatically and are saved in your S3 bucket.
Customer-owned deployment Terraform provisioned Runner + optimisers HTML + JSON reports Scheduled with EventBridge

This page explains what you need to do, what gets deployed in your AWS account and what you receive after each run.

What you do

Expected time: around 10 to 20 minutes if you already have AWS access for Terraform deployment.

  • 1

    Complete the onboarding form.

    Choose your AWS region, report frequency and the AWS services you want the platform to review.
  • 2

    Download your deployment bundle after approval.

    The ZIP already contains your Terraform files, selected Lambda packages and customer configuration.
  • 3

    Quickly review the settings.

    Check the AWS region, schedule, report bucket and selected services before running Terraform.
  • 4

    Deploy into your AWS account with Terraform.

    Terraform creates the runner, enabled optimisers, IAM permissions, schedule, logs and report bucket.
  • 5

    Receive scheduled reports automatically.

    Each run checks your selected services, estimates possible savings and uploads HTML and JSON reports to reports/<customer>/<run_id>.

What Terraform creates

  • One runner Lambda that manages each scheduled report run
  • Only the optimiser Lambdas you selected in the form
  • An IAM role with the permissions needed to collect findings
  • CloudWatch logs for troubleshooting
  • An EventBridge schedule for automatic execution
  • An S3 bucket where reports are stored

Services you can include

Amazon S3 Looks for data that could move to a lower-cost storage class.
Amazon RDS Checks whether database instances appear larger than necessary.
Amazon ECS Reviews container services for potential over-provisioning.
Amazon EKS Checks whether Kubernetes node capacity may be reduced safely.
EC2 Spot Instances Highlights workloads that may be suitable for lower-cost Spot pricing.

What is in your download

  • Terraform files ready to deploy
  • The runner package and only the optimisers you selected
  • Your customer configuration values
  • Deployment and troubleshooting guides written for customers

Deploy commands

cd <downloaded-bundle>/terraform terraform init terraform apply

Replace <downloaded-bundle> with the folder created when you extract the ZIP, for example org_7-deployment. In most cases, your main values are already filled in for you.

What happens after deployment

  • EventBridge starts the runner on your chosen schedule.
  • The runner checks which services are enabled for your account.
  • Each selected optimiser analyses its own AWS service.
  • The runner combines those results into one report.
  • The report is uploaded to S3 as both HTML and JSON.
What you receive
• An HTML report that is easy to read and share.
• A JSON export if you want to reuse the data elsewhere.
• Both files are stored in S3 under reports/<customer>/<run_id>.html and reports/<customer>/<run_id>.json.
