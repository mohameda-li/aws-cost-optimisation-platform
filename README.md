# AWS Cost Optimisation Platform

A serverless AWS FinOps automation platform designed to detect and reduce cloud cost waste across ECS, EKS, RDS, S3, and Spot Instances. The web application manages customer onboarding and bundle generation, and customers deploy the generated bundle into their own AWS account with Terraform.

## Repository Layout

`flaskAPP/web`
: Flask web application, admin/customer flows, bundle generation, templates, and static assets.

`src/runner`
: Runner Lambda source. This orchestrates enabled optimisers, aggregates totals, and generates reports.

`src/optimisers`
: Optimiser Lambda source packages for S3, RDS, ECS, EKS, and Spot.

`src/data`
: Pricing reference data used when bundles are generated.

`docker`
: MySQL initialisation scripts for local development.

## Local Development

Start the web app and database with Docker Compose:

```bash
docker compose up --build
```

The web app will be available on `http://localhost:5050`.

Copy [.env.example](/Users/mohamedali/Library/CloudStorage/OneDrive-UWEBristol/Year%203/DSP/FinalApp/.env.example) to `.env` if you want to override the default local Docker settings.

## Tests

Run the automated test suite with:

```bash
python3 -m unittest discover -s tests -v
```

Run coverage with:

```bash
python3 -m coverage run -m unittest discover -s tests
python3 -m coverage report
```

The current automated coverage includes:

- unit tests for optimiser cost, recommendation, and savings logic
- runner helper tests
- report generation tests
- bundle generation integration tests
- Flask authentication and forgot password route tests
- a basic web app smoke test
- admin approval and management route tests
- customer bundle download route tests
- shared helper tests for bundle and customer configuration logic


## Bundle Generation

The customer deployment bundle is generated by the Flask app.

The bundle builder uses the following source-of-truth directories:

- `src/runner`
- `src/optimisers`
- `src/data`

When a bundle is generated, the app:

1. copies the Terraform, config, and docs template
2. writes `terraform.tfvars`
3. writes `customer_config.json`
4. packages `runner.zip`
5. packages only the selected optimiser zips
6. copies only the selected pricing data directories

Customers do not need to zip Lambda files manually.

## Generated Customer Bundle

Each generated bundle contains:

- `terraform/`
- `lambdas/`
- `config/`
- `data/`
- `docs/`
- `README.md`

Customers deploy from the bundle with:

```bash
cd customer-deployment/terraform
terraform init
terraform apply
```

## Reports

The runner produces:

- HTML reports
- JSON reports

Reports are uploaded to the configured S3 bucket under:

```text
reports/<customer>/<run_id>.html
reports/<customer>/<run_id>.json
```
